# JavaScript SDK Installation

Add the TrackSource JavaScript SDK to your website to track visitors, capture leads, and attribute conversions to marketing channels.

## Quick Installation (Recommended)

Add this single script tag to the `<head>` section of every page:

```html
<script src="https://events.tracksource.io/sdk/ts.min.js" data-tenant="YOUR_TENANT_ID"></script>
```

<Warning>
  Replace `YOUR_TENANT_ID` with your Tenant ID from the TrackSource dashboard (**Settings** → **General**).
</Warning>

This one-line script will:
- Automatically track page views
- Capture UTM parameters and click IDs (fbclid, gclid, ttclid)
- Generate anonymous visitor IDs
- Auto-populate Typeform and Tally hidden fields

## Alternative: Async Loading

For advanced users who want non-blocking async loading:

```html
<script>
  (function(w,d,s,l,i){w[l]=w[l]||[];w[l].push({'ts.start':
  new Date().getTime(),event:'ts.js'});var f=d.getElementsByTagName(s)[0],
  j=d.createElement(s),dl=l!='tsLayer'?'&l='+l:'';j.async=true;j.src=
  'https://events.tracksource.io/sdk/ts.min.js?id='+i+dl;f.parentNode.insertBefore(j,f);
  })(window,document,'script','tsLayer','YOUR_TENANT_ID');
</script>
```

## NPM Installation (Advanced)

<Note>
  The npm package is coming soon. For now, use the script tag method above which works with all frameworks including React, Vue, and Next.js.
</Note>

For React/Next.js projects, you can wrap the script in a component:

```jsx
// components/TrackSource.jsx
import { useEffect } from 'react';

export function TrackSource({ tenantId }) {
  useEffect(() => {
    if (window.TracksSource) return;

    const script = document.createElement('script');
    script.src = 'https://events.tracksource.io/sdk/ts.min.js';
    script.setAttribute('data-tenant', tenantId);
    script.async = true;
    document.head.appendChild(script);
  }, [tenantId]);

  return null;
}

// Usage in App.jsx or layout.tsx
<TrackSource tenantId="YOUR_TENANT_ID" />
```

## Verify Installation

### Method 1: Browser Console

1. Open your website
2. Open Developer Tools (F12 or Cmd+Option+I)
3. Go to the Console tab
4. Look for `[TrackSource] Initialized`

### Method 2: Network Tab

1. Open Developer Tools
2. Go to the Network tab
3. Refresh the page
4. Search for `ts.js`
5. Verify it loads successfully (200 status)

### Method 3: TrackSource Dashboard

1. Go to your TrackSource dashboard
2. Navigate to **Analytics** → **Live Events**
3. Visit your website in another tab
4. See the `page_view` event appear in real-time

## What Gets Tracked Automatically

Once installed, TrackSource automatically tracks:

| Event | When | Data Captured |
|-------|------|---------------|
| `page_view` | Every page load | URL, referrer, title |
| `session_start` | New session begins | UTM parameters, click IDs |

Additional data captured:

- **UTM Parameters**: source, medium, campaign, content, term
- **Click IDs**: fbclid, gclid, ttclid, li_fat_id
- **Browser IDs**: fbp (Facebook), _ga (Google Analytics)
- **Device Info**: Browser, OS, device type
- **Geo Info**: Country, region (from IP)

## Script Loading

The TrackSource SDK is:

- **Async**: Doesn't block page rendering
- **Lightweight**: < 10KB gzipped
- **Cached**: Uses CDN with long cache headers
- **Reliable**: 99.99% uptime SLA

### Loading Order

For best results, load the TrackSource script before other marketing scripts:

```html
<head>
  <!-- TrackSource first -->
  <script>/* TrackSource script */</script>

  <!-- Then other scripts -->
  <script>/* Facebook Pixel */</script>
  <script>/* Google Analytics */</script>
</head>
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Script not loading">
    1. Check browser console for errors
    2. Verify Tenant ID is correct
    3. Check if ad blockers are blocking the script
    4. Ensure script is in the `<head>` section
  </Accordion>

  <Accordion title="Events not appearing in dashboard">
    1. Wait 30 seconds after page load
    2. Check the correct time range in dashboard
    3. Verify Tenant ID matches between script and dashboard
    4. Check Network tab for successful event requests
  </Accordion>

  <Accordion title="Console errors">
    Common errors:

    - `ts is not defined`: Script hasn't loaded yet. Use `window.ts` or wait for load.
    - `Invalid tenant ID`: Check your Tenant ID is correct.
    - `Network error`: Check internet connection and firewall settings.
  </Accordion>

  <Accordion title="Slow page load">
    The script is async and shouldn't slow down page load. If experiencing issues:

    1. Check if other scripts are blocking
    2. Verify CDN is accessible from your region
    3. Contact support if issues persist
  </Accordion>
</AccordionGroup>
