## Base URL

All API requests should be made to:

```
https://events.tracksource.io
```

For self-hosted instances, replace with your domain.

## Authentication

All API endpoints require a **Tenant ID** passed in the header:

```bash
X-Tenant-ID: your_tenant_id
```

<Warning>
Keep your Tenant ID secure. Anyone with this ID can send events to your account.
</Warning>

## Rate Limits

| Endpoint | Rate Limit |
|----------|-----------|
| `/v1/track` | 1000 req/min |
| `/v1/batch` | 100 req/min |
| `/leads/*` | 500 req/min |
| `/v1/events/*` | 100 req/min |

Rate limit headers are included in responses:

```
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 999
X-RateLimit-Reset: 1705320000
```

## Response Format

All responses are JSON:

### Success Response

```json
{
  "success": true,
  "event_id": "evt_abc123",
  "timestamp": "2024-01-15T10:30:00Z"
}
```

### Error Response

```json
{
  "error": "Invalid event data",
  "details": ["event_type is required"],
  "status": 400
}
```

## Common Status Codes

| Code | Description |
|------|-------------|
| `200` | Success |
| `400` | Bad Request - Invalid parameters |
| `401` | Unauthorized - Missing or invalid Tenant ID |
| `429` | Rate Limited - Too many requests |
| `500` | Server Error |

## WebSocket API

For real-time event streaming:

```javascript
const ws = new WebSocket('wss://events.tracksource.io/ws?tenant_id=YOUR_TENANT_ID');

ws.onopen = () => {
  console.log('Connected');

  // Subscribe to specific event types (optional)
  ws.send(JSON.stringify({
    type: 'subscribe',
    event_types: ['purchase', 'lead']
  }));
};

ws.onmessage = (event) => {
  const data = JSON.parse(event.data);
  console.log('Event received:', data);
};
```
