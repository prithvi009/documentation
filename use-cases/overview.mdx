## The Universal Pattern

Regardless of your business type, TrackSource integration follows the same core pattern:

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        CUSTOMER JOURNEY                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│   Ad Click  →  Landing Page  →  Lead Capture  →  Purchase/Conversion    │
│      │             │                │                    │               │
│      ▼             ▼                ▼                    ▼               │
│   ┌─────┐    ┌──────────┐    ┌───────────┐      ┌─────────────┐        │
│   │Click│    │TrackSource│    │  Webhook  │      │   Webhook   │        │
│   │ ID  │    │  Script   │    │(Form/CRM) │      │  (Payment)  │        │
│   └─────┘    └──────────┘    └───────────┘      └─────────────┘        │
│      │             │                │                    │               │
│      └─────────────┴────────────────┴────────────────────┘               │
│                              │                                           │
│                              ▼                                           │
│                    ┌──────────────────┐                                 │
│                    │   TrackSource    │                                 │
│                    │   Attribution    │                                 │
│                    └──────────────────┘                                 │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

### Step 1: Capture Traffic Source

TrackSource automatically captures:
- **UTM Parameters**: `utm_source`, `utm_medium`, `utm_campaign`
- **Click IDs**: `fbclid` (Facebook), `gclid` (Google), `ttclid` (TikTok)
- **Referrer**: Where the visitor came from
- **Landing Page**: First page visited

### Step 2: Link to Identity

When you know who the visitor is (form submit, signup, purchase):
- Pass `ts_anonymous_id` via hidden field OR
- Match by email address

### Step 3: Track Revenue

Connect payment webhooks to attribute revenue to marketing channels.

## Common Challenges Solved

<AccordionGroup>
  <Accordion title="Iframe forms can't read URL parameters">
    When forms are embedded as iframes (like Typeform in GHL), they can't access the parent page's URL. Solution: Pass `ts_anonymous_id` to the iframe URL. See the [Courses guide](/use-cases/courses) for details.
  </Accordion>

  <Accordion title="Multi-step funnels lose attribution">
    When customers move through multiple pages/domains, attribution can be lost. Solution: TrackSource uses first-party cookies and click IDs that persist across pages.
  </Accordion>

  <Accordion title="Offline conversions (phone sales)">
    When sales happen offline or via phone, you need to manually log them. Solution: Use the API to track conversions server-side with the customer's email.
  </Accordion>

  <Accordion title="Long sales cycles">
    B2B and high-ticket sales can take weeks or months. Solution: TrackSource's default 30-day attribution window can be extended, and the full journey is preserved.
  </Accordion>
</AccordionGroup>
